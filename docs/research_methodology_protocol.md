# Research methodology protocol

The methodology separates public benchmark diagnostics, parser validation, conditional L3 evidence and controlled mechanics validation.

FI-2010 is treated as a public LOB feature and short-horizon mid-price movement benchmark. LOBSTER samples are treated as parser and reconstruction validation inputs where files are available. Databento MBO or equivalent L3 data is treated as a conditional order-level route only when access, order IDs and schema checks are configured. Controlled fixtures validate mechanics, accounting and edge-case behaviour only.

The methodology remains claim-aware. Public benchmark data, parser samples, conditional L3 data and generated fixtures must not be merged into a single evidence category.
